Each of the samples found in the samples/ directory of the SDK includes binary executable and CMAKE file that can be used to build the sample. If you like, you can use one of these samples as a starting point for building a new application that uses the ZED SDK.


ZED Depth Viewer

The ZED Depth Viewer displays the depth computed from the ZED stereo feeds.

On the left side you can choose between Left View / Right View / Anaglyph View / Gray Scale Difference View/ Side by Side View or Overlay View.

On the right side it displays directly the depth images stored in the GPU, avoiding costly GPU->CPU transfers by using CUDA OpenGL interoperability.

You can either visualize the Disparity Map (without real world depth information), the Depth Map (with real world depth information) or the Confidence Map (for confidence of disparity estimation) . Press d to toggle between those three options.

Toggle between the Sensing Modes with the r – Sensing Mode : RAW - and f keys – Sensing Mode : FULL

You can set the reliability index with the b – Decrease - and n - Increase – keys

Call the different views using keys 0-5 on your keyboard.

Keyboard Shortcuts

Main Hotkeys

  • b: Decrease Reliability Index by 10
  • n: Increase Reliability Index by 10
  • a: Recompute Alignment
  • r: Sensing Mode : RAW
  • f: Sensing Mode : FULL

Display Hotkeys

  • d: Toggle Disparity / Depth / Confidence view
  • 0: Left View
  • 1: Right View
  • 2: Anaglyph View
  • 3: Gray Scale Difference View
  • 4: Side by Side View
  • 5: Overlay View

ZED 3D Visualization

The ZED 3D Visualizer displays a point cloud visualization of ZED depth by assigning a 3D coordinate to every pixel. On the backend, an OpenGL view displays the computed 3D point cloud. We show how to use a classic Point Cloud class as well as a classic OpenGL render window class.

The 3D origin is represented by a 3D avatar of the ZED Camera.

To rotate around the origin press and hold your left mouse key while moving the mouse. To translate the origin press and hold your right mouse key while moving the mouse. To zoom in and out scroll with your center mouse wheel. To fine-tune your zoom, press and hold your center mouse wheel while you scroll.

You can change the size of the points by pressing the w (smaller) and x (bigger) keyboard keys.

Reset to the default viewpoint by pressing the o key. Switch to the top view by pressing the t key.

Toggle between the Sensing Modes with the r – Sensing Mode : RAW - and f keys – Sensing Mode : FULL

You can set the reliability index with the b – Decrease - and n - Increase - keys

Keyboard Shortcuts

Main Hotkeys

  • b: Decrease Reliability Index by 1
  • n: Increase Reliability Index by 1
  • r: Sensing Mode : RAW
  • f: Sensing Mode : FULL

Display Hotkeys

  • o: Reset Origin View
  • t: Top View
  • w: Decrease Point Size
  • x: Increase Point Size

ZED with OpenCV

The ZED with OpenCV sample shows how to interface the ZED SDK with the OpenCV library. You will find instructive comments within the sample source code. You can either visualize the Disparity Map (without real world depth information) or the Depth Map (with real world depth information). Press d to toggle between the two.

Call the different views using keys 0-5 on your keyboard.

Press s to view the confidence map.

Toggle between the Sensing Modes with the r – Sensing Mode : RAW - and f keys – Sensing Mode : FULL

You can set the reliability index with the b – Decrease - and n - Increase - keys

You can save a PNG image of the disparity by pressing the v key or an SBS screenshot using the w key.

Keyboard Shortcuts

Main Hotkeys

  • b: Decrease Reliability Index by 1
  • n: Increase Reliability Index by 1
  • r: Sensing Mode : RAW
  • f: Sensing Mode : FULL
  • s: Display Confidence Map
  • w: Save Side by Side image in PNG
  • v: Save Disparity image in PNG

Display Hotkeys

  • d: Toggle Disparity / Depth view
  • 0: Left View
  • 1: Right View
  • 2: Anaglyph View
  • 3: Gray Scale Difference View
  • 4: Side by Side View
  • 5: Overlay View

Building the Samples

To build a sample, a very simple way to do it is to use CMAKE with the CMakeLists.txt we provide in command line :

  • Create a build directory by calling mkdir build
  • Enter the build directory by calling cd build
  • Create the VS project (Windows) or the Makefile (Linux) by calling cmake ..
  • On Linux, just call make to build the sample
  • On Windows, Open the VS project to compile the sample.
